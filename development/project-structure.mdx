## Directory descriptions

**`components/`** — Reusable UI components not tied to a specific route. Currently contains `gym-classes-component`, which renders the list of available gym classes and is shared between the home page and admin views.

**`guard/`** — Angular functional route guards. `HomeGuard` runs on the `/home` route and redirects authenticated users to their appropriate dashboard (`/admin-dashboard` for admins, `/user-dashboard` for regular users).

**`interfaces/`** — TypeScript interfaces that describe all domain objects exchanged with the backend API. These are used as generic type parameters in service methods and component properties throughout the app.

**`pages/`** — Routed view components. Each subdirectory maps to one or more application routes. The `auth/` subdirectory contains the login and registration flows. The `dashboard/` subdirectory contains separate views for users and admins.

**`services/`** — Angular `@Injectable` services that encapsulate all HTTP communication with the backend. Each service corresponds to a specific resource (bookings, schedules, classes, trainers, users). Auth lives in its own `auth/` subdirectory because it additionally manages client-side session state via a `BehaviorSubject`.

**`shared/`** — Layout components rendered in every view: the navigation bar, sidebar menu, modal dialog, and footer. These are declared at the module level and do not belong to any single page.

**`styles.css`** — The global stylesheet. It imports Tailwind CSS 4, which is processed through PostCSS via `@tailwindcss/postcss`.

## Application routes

Routes are defined in `src/app/app-routing-module.ts` and registered with `RouterModule.forRoot`. Unmatched paths redirect to `/home` via a wildcard catch-all.

| Path | Component | Guard |
|---|---|---|
| `/home` | `HomeComponent` | `HomeGuard` |
| `/login` | `LoginComponent` | — |
| `/create-acount` | `CreateAcountComponent` | — |
| `/user-dashboard` | `DashboardUserComponent` | — |
| `/admin-dashboard` | `DashboardAdminComponent` | — |
| `/dashboard/users` | `UserComponent` | — |
| `/dashboard/horarios` | `ScheduleComponent` | — |
| `/dashboard/reservas` | `BookingComponent` | — |
| `/create-admin` | `CreateAdminComponent` | — |
| `**` | — | redirects to `/home` |

## HomeGuard behavior

`HomeGuard` is a functional `CanActivateFn` guard. It subscribes to the `AuthService` user observable. If the current user has roles assigned, the guard redirects them away from `/home` to their dashboard and blocks access to the route. If no session data is present, access to `/home` is permitted.

```typescript
// src/app/guard/home-guard.ts
export const HomeGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);
  let isUserLoggedIn = false;

  authService.getUserLoggedIn$().subscribe((userData: CurrentAuthUser) => {
    if (userData && userData.roles && userData.roles.length > 0) {
      if (userData.roles.includes('ADMIN')) {
        router.navigate(['/admin-dashboard']);
      } else {
        router.navigate(['/user-dashboard']);
      }
      isUserLoggedIn = true;
    } else {
      isUserLoggedIn = false;
    }
  });

  return !isUserLoggedIn;
};
```

<Info>
  The guard returns `!isUserLoggedIn` synchronously while the observable resolves. Unauthenticated users always reach `/home`; authenticated users are immediately redirected to their dashboard.
</Info>
