This guide walks through the core member flow: creating an account, logging in, finding a class, booking a slot, and reviewing your bookings.

<Info>
  The FitWave backend must be running at `http://localhost:8080` before you start the Angular dev server. The frontend sends all API requests to that origin.
</Info>

## Start the development server

```bash
npm install
npm start
```

The app is served at `http://localhost:4200` by default.

## Member flow

<Steps>
  <Step title="Register an account">
    Navigate to `/create-acount`. Fill in your first name, last name, email, and password.

    The form submits a `POST` request to `/api/auth/user-register`. On success, you are signed in automatically and redirected to your dashboard — no separate login step needed.

    ```typescript
    // CreatedUser payload sent on registration
    {
      firstName: string;
      lastName: string;
      email: string;
      password: string;
      roles: string[];
    }
    ```
  </Step>

  <Step title="Log in">
    Navigate to `/login`. Enter the email and password you registered with.

    A `POST` request is sent to `/api/auth/user-auth`. The server responds with a session cookie and your user profile including your `roles` array.

    - Members with the `ROLE_USER` role are redirected to `/user-dashboard`.
    - Accounts with an admin role are redirected to `/admin-dashboard`.
  </Step>

  <Step title="Browse classes">
    From `/user-dashboard`, the app loads all available gym classes via `GET /api/classes`. Each class card shows:

    - Class title and description
    - Difficulty level
    - Assigned trainer name

    Select a class to see its scheduled time slots.
  </Step>

  <Step title="Book a class">
    Choose an available schedule slot. The app checks remaining capacity using `occupiedSlots` vs `totalSlots` from the schedule.

    Confirming a booking sends a `POST` request to `/api/bookings` with your user ID, the selected schedule ID, and an initial status of `PENDING`.

    ```typescript
    // Booking payload
    {
      userId: number;
      scheduleId: number;
      status: 'PENDING';
    }
    ```
  </Step>

  <Step title="View and manage bookings">
    Your bookings are listed on `/user-dashboard` after a `GET /api/bookings/:userId` call. Each booking shows the class, schedule time, and current status (`PENDING`, `CONFIRMED`, or `CANCELLED`).

    To cancel a booking, select it and confirm. This sends `PATCH /api/bookings/:id/cancel` to the backend and updates the slot's occupied count.
  </Step>
</Steps>

<Tip>
  Administrators have access to `/admin-dashboard`, which adds full CRUD for gym classes (`/api/classes`), trainers (`/api/trainers`), and schedules (`/api/schedules`), as well as user management at `dashboard/users` — including the ability to disable member accounts via `/api/user/disable-user/:email`.
</Tip>
